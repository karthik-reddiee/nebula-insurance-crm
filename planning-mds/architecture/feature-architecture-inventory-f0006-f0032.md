# Feature Architecture Inventory: F0006-F0032

This inventory summarizes the primary architecture and solution-design additions introduced by the planned feature set from `F0006` through `F0032`.

Traceability labels used in feature PRDs are defined in [Feature Architecture Traceability Taxonomy](./feature-architecture-traceability-taxonomy.md).

| Feature | Primary Domain / Design Addition | Key Components Introduced | Workflow / State Model | Integration / Runtime Notes | Security / Operational Focus | Primary ADRs |
|---------|----------------------------------|---------------------------|------------------------|-----------------------------|------------------------------|--------------|
| F0006 | Submission intake becomes a first-class domain workflow | Submission aggregate, completeness policy, intake workflow service, queue handoff adapter | Intake states and append-only workflow transitions | Document completeness linkage and downstream assignment events | ABAC on submission actions, audit trail, pageable intake queries | ADR-011, ADR-012 |
| F0007 | Renewal operating model and time-based orchestration | Renewal aggregate, reminder scheduler, renewal worklists, escalation handler | Renewal lifecycle plus Temporal-driven reminders and escalations | Consumes policy expirations and emits tasks or queue signals | Workflow observability, idempotent reminders, overdue metrics | ADR-010, ADR-011 |
| F0008 | Analytical broker performance layer | Broker scorecards, benchmark service, trend projections, materialized metrics | Read-model oriented, no primary transactional workflow | Reuses workflow and activity history for metrics | Hierarchy-aware visibility, refresh cadence, query cost control | ADR-014 |
| F0016 | Account-centered CRM root aggregate | Account aggregate, 360 composer, account summaries, relationship context services | Account itself is not a heavy state machine; focuses on related-context composition | Stable account APIs reused by policy, submission, and service features | PII-sensitive account access, related-list performance, merge safety | None currently required |
| F0017 | Broker hierarchy and producer ownership structure | Hierarchy service, territory rules, producer ownership model, rollup projections | Effective-dated ownership and hierarchy changes | Structural context feeds routing, reporting, and visibility | Hierarchy-aware authorization, change audit, rollup recomputation | ADR-013, ADR-014 |
| F0018 | Policy becomes a first-class lifecycle record | Policy aggregate, endorsement/version services, policy timeline, renewal linkage | Policy lifecycle distinct from submission workflow | Emits policy lifecycle events and anchors downstream renewals and billing | High-sensitivity policy access, concurrency on versioned updates | ADR-010, ADR-011, ADR-012 |
| F0019 | End-to-end quote, approval, and bind workflow | Quoting service, approval component, proposal package composer, bind decision handler | Expanded submission state machine with human approval checkpoints | Document linkage, downstream policy handoff, workflow events | Approval authority checks, maker-checker separation, stuck-state metrics | ADR-011, ADR-012 |
| F0020 | Shared document system of record | Document service, metadata store, version manager, entity linkage layer | Document versioning and completeness state | Storage abstraction, parent-record linkage, future OCR hook points | Parent-based access control, file validation, audit of document actions | ADR-012 |
| F0021 | Structured communication capture in CRM | Communication service, follow-up linkage, timeline composer, activity records | Activity and follow-up linkage rather than a standalone workflow engine | Later connectors can map into the same communication model | Sensitive-note access, duplicate ingestion control, timeline growth | ADR-015 |
| F0022 | Explicit operational routing and queueing | WorkQueue, WorkQueueMember, AssignmentRule, CoverageWindow, QueueWorkItem, RoutingDecisionLog, routing source adapters | Deterministic routing and coverage rules with auditable outcomes | Consumes task/submission/renewal route-eligible events and can later be governed via F0032 without replacing F0022 tables | Manager/admin reassignment powers, source-record ABAC intersection, rule observability, no-match fallback queue | ADR-013 (Accepted), ADR-016 |
| F0023 | Shared search and operational reporting layer | Search index, saved-view service, reporting projections, workload dashboards | Reporting on workflow state and aging rather than owning workflow | Async indexing and report projections across major entities | Auth-aware search, index lag monitoring, query governance | ADR-014, ADR-016 |
| F0024 | Post-bind service-case operating model | ServiceCase aggregate, service workflow service, claim-reference model, case timeline | Case lifecycle for service intake through resolution | Reuses communication capture and links to account or policy context | Service-team access control, backlog and SLA tracking | ADR-011 |
| F0025 | Revenue-awareness and commission attribution | Commission rules service, split engine, revenue records, producer or carrier rollups | Effective-dated split and commission records | Downstream handoff to billing and reconciliation, no full ledger | Financial audit, override control, recalculation cost management | None currently required |
| F0026 | Finance-adjacent billing operations layer | Invoice service, billing-event records, reconciliation workflow, exception views | Billing and reconciliation statuses separate from GL | Batch import or export friendly, hooks from policy and commission flows | Finance-only access, batch idempotency, exception aging visibility | ADR-011, ADR-015 |
| F0027 | Outbound insurance document generation pipeline | Template engine, merge-data assembler, render service, generated artifact audit | Draft-to-issued document output lifecycle | Generated artifacts stored via document system, e-sign later | Template governance, render validation, output traceability | ADR-012, ADR-016 |
| F0028 | Carrier and market relationship model | Carrier aggregate, appetite-note service, appointment context, market activity feed | Effective-dated market intelligence and relationship state | Links into submission and reporting flows, no rating engine yet | Sensitive market-intelligence access, freshness and provenance controls | ADR-014 |
| F0029 | External broker collaboration boundary | Portal auth mapping, broker-scoped API/BFF, external document exchange, collaboration actions | Portal exposes a safe subset of internal workflow states | Depends on integration hub for notifications and external exchange evolution | Strong tenant isolation, rate limiting, external audit visibility | ADR-006, ADR-008, ADR-012, ADR-015 |
| F0030 | Cross-system integration platform boundary | Connector adapters, canonical contracts, outbox pattern, monitoring and replay controls | Integration sync states, retries, dead-letter handling | Webhooks, batch exchange, async delivery, connector health | Secrets management, contract versioning, idempotent processing | ADR-015, ADR-016 |
| F0031 | Migration and import execution pipeline | Import jobs, staging records, validation services, dedup and merge review | Job lifecycle for upload, validate, review, commit | Async batch processing, one-way migration adapters | Elevated-role access, lineage tracking, sensitive file handling | None currently required |
| F0032 | Governed runtime configuration surface | Config registry, reference-data service, publish and validation flow, admin console APIs | Draft, validated, and published config states | Consumed by queues, templates, reporting, and future configurable modules | Admin-only change control, publish audit, drift and refresh monitoring | ADR-013, ADR-014, ADR-016 |
