# Persona: Underwriter

**Role:** Underwriter
**Priority:** Secondary

## Context

Reviews submissions, policies, and renewals that move through underwriting stages. Needs status, owner, and due-work context without becoming the owner of relationship management reports.

## Goals

- Search underwriting-relevant records by account, broker, policy, submission, or renewal identifiers.
- Filter to work in underwriting stages and open the source workflow screen.
- See aging and due-work context for items awaiting underwriting action.

## Pain Points

- Underwriting work can be hidden behind relationship-manager views or entity-specific filters.
- Similar account or policy names increase the risk of opening the wrong record.
- Aging and due-work context is hard to compare when each workflow page has a different filter surface.

## Jobs To Be Done

- When I receive a follow-up request, I want one search path to find the submission, renewal, or policy so I can update the correct workflow.
- When reviewing my queue, I want a saved underwriting view for work in underwriting statuses so I can prioritize records needing action.
- When work is aging, I want to drill into the source record so I can complete the next workflow step.

## Success Measures

- Underwriting-stage filters are available where the source workflow exposes status.
- Due and aging reports identify records in underwriting-owned stages.
- Results respect role and resource authorization before display.
