# Persona: Relationship Manager

**Role:** Broker Relationship Manager
**Priority:** Primary

## Context

Owns broker and account relationships and needs fast context before calls, renewals, and follow-up. Works across broker, account, policy, submission, renewal, task, and timeline surfaces.

## Goals

- Find the right broker, account, policy, submission, renewal, or task from one search entry.
- Reuse personal views for common relationship follow-up patterns.
- Navigate from a search result directly to the authoritative 360 or workflow screen.

## Pain Points

- Similar broker/account names require manual checking across multiple pages.
- Repeated filters for "my open follow-up" or "accounts with active renewals" are rebuilt each session.
- Search results that lack status, owner, or route context do not answer whether the record is the right one.

## Jobs To Be Done

- When preparing for a broker call, I want to search by broker, account, policy number, submission number, or renewal reference so I can open the correct source record.
- When monitoring my book, I want saved personal views so I can return to the same filtered work list after reload.
- When a search term matches multiple object types, I want grouped results with status and owner context so I can select the right record.

## Success Measures

- Search results are grouped and deep-link to source screens.
- Personal saved views persist after reload and can be updated or deleted.
- Unauthorized records never appear in results or counts.
