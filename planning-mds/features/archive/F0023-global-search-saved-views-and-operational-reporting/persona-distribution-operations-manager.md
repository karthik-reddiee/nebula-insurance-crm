# Persona: Distribution Operations Manager

**Role:** Distribution Operations Manager
**Priority:** Primary

## Context

Manages daily commercial P&C distribution work across submissions, renewals, policy follow-up, broker relationships, and task queues. Reviews workload and aging several times per day and needs a shared view of what needs attention.

## Goals

- Find active work across broker, account, policy, submission, renewal, and task records without switching between entity-specific pages.
- Identify overdue, aging, and owner-heavy work before SLA or customer follow-up misses occur.
- Standardize team views for daily triage without maintaining separate spreadsheets.

## Pain Points

- Operational questions require repeated filtering across multiple list pages.
- Team members use different filter combinations, so daily standups do not start from a common view.
- Counts and due-work lists are often rebuilt manually, which creates stale or inconsistent reporting.

## Jobs To Be Done

- When planning the morning workload, I want a saved team view of due and overdue work so I can assign follow-up from a common operating picture.
- When a broker asks for status, I want to search across related records so I can find the active submission, renewal, policy, or task quickly.
- When backlog grows, I want workflow aging drilldowns so I can focus on records stuck beyond threshold.

## Success Measures

- Daily triage can start from a saved team view.
- Due-work and aging reports identify the source records that need action.
- Search and report counts match the user's permission scope.
